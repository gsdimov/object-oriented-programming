## Работа с потоци

- полиморфизъм при поточни типове

```cpp
// `ostream`, а не - `ofstream`! искаме функцията да работи за произволен
// поточен тип за писане
void writeArr(int *arr, size_t size, std::ostream &os=std::cout) {
  if (size == 0)
    return;

  for (size_t i = 0; i < size-1; ++i) {
    os << arr[i] << ' ';
  }
  os << arr[size-1];
}
```

```cpp
int arr[4]{1,2,3,4};
// пишем елементите на стандартния изход
writeArr(arr, 4);
writeArr(arr, 4, std::cout);

std::ofstream ofs(filename, std::ios::out);
if (!ofs.is_open()) return -1;

// пишем елементите в текстов файл
writeArr(arr, 4, ofs);

ofs.close();
```

- валидност на поток:

```cpp
if (std::cout.good()) { /*...*/ }
if (std::cout) { /*...*/ }
```

## Стандартен вход/изход

- `std::cin`;

- `std::cout`;

- `std::clog` - буфериран поток за стандартен изход. Използване се за натрупване
  на информация за някакви извършени операции (logging);

- `std::cerr` - небуферирран поток за стандартен изход (т.нар. стандартен изход за грешки).
  Използва се за записване на настъпили грешки и тъй като е небуферирран, те се извеждат
  веднага.

## Буферирани и небуферирани потоци

При буферираните потоци данните, които искаме да запишем, се натрупват в RAM паметта и се
записват в самия поток едва при достигане на определен лимит или при ръчно форсиране на такова
записване. При небуферирираните потоци е точно обратното - данните директно се записват в потока.
Това има смисъл за потоци за грешки като `std::cerr`, където грешката може да бъде фатална и да искаме
да разберем за нея възможно най-рано.

- `std::flush` - форсира извеждането на данните в конкретния поток:

  ```cpp
  std::cout << "this message will be written to the standard output stream immediately";
  std::cout << std::flush;
  ```

`std::endl` не само извежда символ за нов ред, но изпълнява функционалността и на `std::flush`.

## Текстови файлове

### Четене от текстов файл (`std::ifstream`)

  - отваряне на текстов файл в режим на четене, използвайки флага `std::ios::in`:

    ```cpp
    // filename е някакъв низ
    std::ifstream ifs(filename, std::ios::in);
    if (!ifs.is_open()) {
      // сигнализираме за грешка по някакъв начин - напр. невалидна стойност
    }
    // логика...
    ifs.close(); // затваряме файла
    ```

    ```cpp
    std::ifstream ifs; // тук не сме отворили файл!
    ifs.open(filename, std::ios::in);
    if (!ifs.is_open()) {
      // сигнализираме за грешка
    }
    // логика...
    ifs.close();
    ```

  - четене на съдържанието на файл (приемаме, че сме отворили и затворили файла, както е показано по-горе):

    ```cpp
    // четем файла символ по символ. get(char) мести указателя за четене с една позиция напред
    char c;
    while (ifs.get(c)) {
      std::cout << c;
    }
    ```

    ```cpp
    // четене в символен буфер с getline(char *buf, size_t sz, char delim='\n')
    char buf[1024];
    while (ifs.getline(buf, 1024)) {
      std::cout << buf;
    }
    ```

    ```cpp
    // четене в променлива от произволен тип (ad hoc полиморфизъм);
    // за разлика от предните 2 пропуска whitespace-ове (' ', '\t', '\n' и др.)
    int sum = 0, x;
    while (ifs >> x) {
      sum += x;
    }
    ```

    ```cpp
    // ГРЕШНО Е ДА НАПИШЕМ СЛЕДНОТО:
    while (ifs.get()) {
      // ...
    }
    // тъй като istream.get() връща char, а не - самия поток (себе си),
    // така че оценяваме символа до булева стойност
    ```

### Файлов указател за четене (get указател)

  - местене на файлов указател - `seekg`

    ```cpp
    // на 0 позиции след началото на файла
    ifs.seekg(0, std::ios::beg);

    // на 0 позиции след края на файла
    ifs.seekg(0, std::ios::end);

    // с една позиция напред спрямо текущата позиция
    ifs.seekg(1, std::ios::cur);

    // с една позиция назад спрямо текущата позиция
    ifs.seekg(-1, std::ios::cur);
    ```

  - извличане на позицията на файлов указател - `tellg`

    ```cpp
    // големина на файл
    ifs.seekg(0, std::ios::end);
    long long size = ifs.tellg();
    ```

### Писане в текстов файл (`std::ofstream`)

  - отваряне на файл в различни режими (можем и с метода `open`):

    ```cpp
    // ако файлът не съществува, създава нов такъв;
    // в противен случай опитва да отвори съществуващ файл,
    // като ИЗТРИВА цялото му съдържание
    std::ofstream ofs(filename, std::ios::out);
    if (!ofs.is_open()) { /*...*/ }
    // логика
    ofs.close();
    ```

    ```cpp
    // ако файлът не съществува, отваря нов такъв;
    // в противен случай опитва да отвори съществуващ файл,
    // като НЕ изтрива съдържанието му, и мести указателя за писане
    // в края на файла. писането става САМО в края на файла!
    std::ofstream ofs(filename, std::ios::out | std::ios::app);
    if (!ofs.is_open()) { /*...*/ }
    // логика
    ofs.close();
    ```

    ```cpp
    // ако файлът не съществува, отваря нов такъв;
    // в противен случай опитва да отвори съществуващ файл,
    // като НЕ изтрива съдържанието му, и мести указателя за писане
    // в края на файла. писането става в позицията на put указателя
    std::ofstream ofs(filename, std::ios::out | std::ios::ate);
    if (!ofs.is_open()) { /*...*/ }
    // логика
    ofs.close();
    ```

  - писане в текстов файл:

    ```cpp
    // записване на единичен символ
    ofs.put('a');
    ofs.put('b').put('c');
    ```

    ```cpp
    // можем да записваме данни от различни типове,
    // без да се налага да ги преобразуваме до низове,
    // тъй като operator<< е пренатоварен (overloaded)
    ofs << 'a';
    ofs << "bc";
    ofs << " " << 42;
    ```

### Файлов указател за писане (put указател)

  Писането във файл чрез `put`, `operator<<` и т.н. **променят** позицията на файловия указател за писане.

  - местене на файлов указател - `seekp`

    ```cpp
    std::ofstream ofs(filename, std::ios::out);
    if (!ofs.is_open()) { /* сигнализираме за грешка */ }

    ofs << "Rorem ipsum dolor sit amet";

    // местим put указателя обратно в началото на файла
    ofs.seekp(0, std::ios::beg);
    ofs.put('L');

    ofs.close();
    ```

    ```cpp
    // std::ios::trunc изтрива съдържанието на съществуващ файл
    std::ofstream ofs(filename, std::ios::out | std::ios::app | std::ios::trunc);
    if (!ofs.is_open()) { /*...*/ }

    ofs << "Rorem ipsum dolor sit amet";

    // местим указателя за писане в началото на файла
    ofs.seekp(0, std::ios::beg);
    // но продължаваме да пишем в неговия край въпреки това!
    ofs.put('L'); // не променя Rorem в Lorem!

    ofs.close();
    ```

  - извличане на позицията на файлов указател - `tellp`

    ```cpp
    std::ofstream ofs(filename, std::ios::out | std::ios::app);
    if (!ofs.is_open()) { /*...*/ }

    // връща големината на файла при std::ios::app или std::ios::ate
    std::cout << ofs.tellp() << '\n';

    ofs.seekp(0, std::ios::beg);
    // връща 0, но писането става само и единствено в края!
    std::cout << ofs.tellp() << '\n';

    ofs.close();
    ```

### Едновременно писане и четене от текстов файл

В общия случай едновременното писане и четене от файл е сравнително по-неприятно
за правене и няма особен смисъл, така че, ако е възможно, бихме искали да го избягваме.

Използваме обекти от тип `std::fstream`:

```cpp
// не се изтрива съдържанието на файла!
std::fstream fs(filename, std::ios::in | std::ios::out);
if (!fs.is_open()) { /*...*/ }

fs.seekg(0, std::ios::end);
fs.seekp(0, std::ios::end);

fs << 42;

int x;
fs >> x; // четем 42

fs.close();
```

## Двоични файлове

Двоичните файлове могат да бъдат отваряне и в режим на писане, и в режим на четене.
Горните неща за текстови файлове все още важат, но за отваряне в двоичен режим ще
използваме още един флаг - `std::ios::binary`. Няма да използваме горните функции/оператори
за четене от двоични файлове, вместо това ще използваме методите `read` и `write`.

---

### Накратко за reinterpret_cast

Досега сме виждали два начина за преобразуване на един тип данни до друг:

- C-style casting:

  ```cpp
  int x = 10, y = 3;
  double res = (double)x / (double)(y + 3);
  ```

- `static_cast` (по-безопасен вариант на горното):

  ```cpp
  int x = 10, y = 3;
  double res = static_cast<double>(x) / static_cast<double>(y + 3);
  ```

`reinterpret_cast` е начин за преобразуване на типа данни, към който сочи даден указател:

- C-style casting:

  ```cpp
  int *arr = new int[10];
  double *darr = (double *)arr;
  ```

- `reinterpret_cast` (по-безопасен вариант на горното):

  ```cpp
  int *arr = new int[10];
  double *darr = reinterpret_cast<double *>(arr);
  ```

### Записване на обекти без динамична памет в двоичен файл

Работи само ако обектът няма указатели/динамична памет.

```cpp
struct Point {
  double x;
  double y;

ofs.write(reinterpret_cast<char const *>(&p), sizeof(p));
```

### Записване на обекти с динамична памет

НЕ можем просто да направим:

```cpp
ofs.write(reinterpret_cast<char const *>(&obj), sizeof(obj)); // ГРЕШНО
```

Ако обектът съдържа указатели, ще запишем адреси, а не - реалните данни.

```cpp
void writeStudent(std::ofstream& ofs, const Student& s) {
  size_t len = std::strlen(s.name);

  // записваме дължината
  ofs.write(reinterpret_cast<char const *>(&len), sizeof(len));

  // записваме символите
  ofs.write(s.name, len);

  // записваме останалите полета
  ofs.write(reinterpret_cast<char const *>(&s.age), sizeof(s.age));
}
```

### Четене от двоичен файл

```cpp
Point p;
ifs.read(reinterpret_cast<char *>(&p), sizeof(p));
```

```cpp
void readStudent(std::ifstream& ifs, Student& s) {
  size_t len;

  // четем дължината
  ifs.read(reinterpret_cast<char *>(&len), sizeof(len));

  // заделяме памет
  s.name = new char[len + 1];

  // четем символите
  ifs.read(s.name, len);
  s.name[len] = '\0';

  // четем останалите полета
  ifs.read(reinterpret_cast<char *>(&s.age), sizeof(s.age));
}
```

## По-сложни примери

### 1. Големина на файл (първа задача от практикума) за вече *отворен* файлов поток

- линейно, т.е. минаваме последователно през всеки символ от файла;
  трябва да правим допълнителни проверки, защото приемаме вече отворен файл,
  който потенциално може да е бил използван от друга процедура (т.е. невалиден поток,
  файлов указател, който да сочи на произволно място и т.н.):

```cpp
long long size(std::ifstream &ifs) {
  // не сме отворили файл или потокът е в невалидно състояние
  if (!ifs.is_open() || !ifs.good())
    return -1;

  // файловият указател за четене може да се не намира в началото
  ifs.seekg(0, std::ios::beg);

  long long cnt = 0;

  char c;
  while (ifs.get(c)) {
    cnt++;
  }

  // за да излезем от цикъла, ifs.get(c) трябва да се оцени до лъжа -
  // това става, ако потокът, който върне методът get, е невалиден,
  // т.е. след приключването на цикъла бихме искали да възстановим валидното състояние
  ifs.clear();

  // не е задължително; нещо друго, което можем да направим,
  // е да запазим предишната позиция на файловия указател (tellg)
  // и да я възстановим тук
  ifs.seekg(0, std::ios::beg);

  return cnt;
}
```

- константно, т.е. местим директно файловия указател в края на файла:

```cpp
long long size(std::ifstream &ifs) {
  if (!ifs.is_open() || !ifs.good())
    return -1;

  // това не прави потока невалиден!
  // указателят ще сочи към EOF, но тъй като не сме направили опит
  // за прочитането му, НЯМА да доведе до невалидно състояние на потока
  ifs.seekg(0, std::ios::end);
  return ifs.tellg();
}
```

## Често срещани грешки

### 1. `!<istream>.eof()` в цикли

```cpp
int x;
while (!ifs.eof()) {
  // или какъвто и да бил друг начин на четене от файла
  ifs >> x;
}
```

Флагът `std::ios::eof` се вдига едва когато *се опитаме* да прочетем
края на файла, а не - когато само го достигнем. Това води до четене на
последната стойност 2 пъти. Затова като условие в циклите използваме
функцията/оператора, които извършват самото четене:

```cpp
int x;
while (ifs >> x) {
  // ...
}
```

### 2. Използване на флаговете за състояние вместо методите

Грешно е да използваме флаговете `std::ios::goodbit`, `std::ios::eofbit` и т.н.
за проверка на състоянието на даден поток. Това са просто константи и техните стойности
не се променят в зависимост от състоянието на някакъв поток (това само по себе си поражда въпроса
какво означава `std::ios::goodbit` да има конкретна стойност за някакъв поток, при положение че можем
да имаме неограничен брой потоци и нямаме конкретен начин да обвържем `std::ios::goodbit` с даден поток).
За да проверим валидността на даден поток, използваме следните *методи* за *обекти* от поточен тип:

| Метод | Описание |
|-------|----------|
| `<stream_type>.good()` | Потокът е във валидно състояние (няма грешки и не е достигнат EOF). |
| `<stream_type>.eof()`  | Достигнали сме края на потока (EOF флагът е зададен). |
| `<stream_type>.fail()` | Възникнала е логическа грешка при четене (напр. опит за четене на `int` от текст), потокът е в невалидно състояние. |
| `<stream_type>.bad()`  | Възникнала е сериозна входно-изходна грешка (напр. хардуерен проблем). |

Състоянието на поток може да се промени ръчно (това се прави, ако има смисъл - напр. в някои ситуации бихме искали
съвсем нарочно да оставим даден поток в невалидно състояние с цел той да не се използва) с метода `clear()`:

```cpp
<stream_type>.clear();
<stream_type>.clear(std::ios::goodbit);
// горните 2 правят едно и също

// тук ръчно правим състоянието на потока невалидно
<stream_type>.clear(std::ios::eofbit | std::ios::failbit);
```

Бихме могли да използваме `setstate` вместо `clear` и `rdstate` + флаг за състояние вместо методите за състояние:

```cpp
<stream_type>.setstate(std::ios::failbit);
if (<stream_type>.rdstate() & std::ios::failbit) {
  // ...
}
```
